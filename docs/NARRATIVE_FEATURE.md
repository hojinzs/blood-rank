# 기능 명세: 축적 데이터 기반 경쟁 내러티브 시스템

**버전:** v0.1 (초안)
**작성일:** 2026-03-10
**상태:** 계획 중
**선행 조건:** 일별 스크래핑 데이터가 최소 2일 이상 축적

---

## 1. 개요

데이터가 쌓일수록 경쟁 서사가 풍부해지는 시스템. 단순 "오늘의 순위"를 넘어, 연속 기록·전일 대비 변화·위기 경보 등으로 혈액형 간 경쟁 내러티브를 자동 생성한다.

### 핵심 가치
- 매일 달라지는 순위 변동 스토리로 재방문 유인
- "3주 연속 꼴찌" 같은 서사가 SNS 공유 소재가 됨
- 위기 상황(2일 미만)에 시각적 긴장감으로 헌혈 동기 강화

---

## 2. 세부 기능

### 2-1. 전일 대비 변화 카피 (Delta)

**데이터 소스:** `blood_supply_latest` 뷰의 `days_delta`, `rank_delta`

| 변화 | 카피 예시 |
|------|----------|
| 순위 상승 | "어제보다 2계단 상승! O형 오늘 열정 폭발 중" |
| 순위 하락 | "어제보다 1계단 하락... A형 오늘 뭐했어요?" |
| 순위 유지 (1위) | "연일 1위 수성 중. 이 정도면 인정" |
| 순위 유지 (꼴찌) | "오늘도 꼴찌. 이쯤 되면 전통인가요?" |
| 보유일수 증가 | "▲ 0.3일 증가 — 누군가 헌혈했군요" |
| 보유일수 감소 | "▼ 0.5일 감소 — 피가 마르고 있어요" |

**선행 조건:** 최소 2일치 데이터 필요. 첫날은 delta 표시 불가 → "아직 비교할 데이터가 없어요" 처리.

---

### 2-2. 연속 순위 기록 (Streaks)

**데이터 소스:** `blood_supply_streaks` 테이블

#### 추적 대상
- **1위 연속 기록:** 며칠 연속 1위를 유지했는지
- **꼴찌 연속 기록:** 며칠 연속 꼴찌인지

#### 표시 카피

| 연속 일수 | 1위 카피 | 꼴찌 카피 |
|-----------|---------|----------|
| 3일 | "3일 연속 1위 🔥 이 기세 실화?" | "3일 연속 꼴찌. 슬슬 걱정되는데요?" |
| 7일 | "7일 연속 1위! 이번 주 완전 정복" | "7일 연속 꼴찌 🔴 일주일째 바닥인데요?" |
| 14일 | "2주 연속 1위. 전설이 되고 있어요" | "2주 연속 꼴찌. 이제 습관이 된 건가요?" |
| 30일+ | "한 달 넘게 1위. 명예의 전당감" | "한 달 넘게 꼴찌. 이건 좀 심각한데요?" |

#### Streak 갱신 로직
```
매일 스크래핑 후:
  1. 오늘 rank 계산
  2. is_current=true인 streak 조회
  3-a. 같은 rank_position이면 → streak_length += 1, streak_end = today
  3-b. 다른 rank_position이면 → 기존 streak 종료 (is_current=false), 새 streak 생성
```

### 2-3. 위기 경보 시스템

**트리거:** 특정 혈액형 보유일수 2일 미만 (`status = 'critical'`)

#### 시각적 변화
| 요소 | 평상시 | 위기 경보 시 |
|------|--------|------------|
| 해당 카드 배경 | 혈액형 고유 색상 | 붉은 톤 강조 + 펄스 애니메이션 |
| 상태 아이콘 | 🟢🟡🔴 | 🚨 깜빡임 효과 |
| CTA 버튼 | "지금 헌혈 예약하기" | "긴급 헌혈 예약하기" (강조 스타일) |
| 카피 톤 | 가벼운 도발 | 긴박감 있는 어조 |

#### 위기 카피 예시
- "🚨 A형 보유량 1.2일. 내일이면 바닥입니다."
- "🚨 긴급! O형 0.8일치 남았어요. 장난 아니에요."

## 3. 데이터 요구사항

### 기존 테이블 활용

#### `blood_supply_latest` 뷰 (이미 존재)
- `days_delta`: 전일 대비 보유일수 증감
- `rank_delta`: 전일 대비 순위 변화 (음수 = 상승)

#### `blood_supply_streaks` 테이블 (생성 필요)
```sql
CREATE TABLE blood_supply_streaks (
  id             uuid     PRIMARY KEY DEFAULT gen_random_uuid(),
  blood_type     text     NOT NULL CHECK (blood_type IN ('A','B','O','AB')),
  rank_position  smallint NOT NULL CHECK (rank_position IN (1, 4)),
  streak_start   date     NOT NULL,
  streak_end     date,
  streak_length  int      NOT NULL DEFAULT 1,
  is_current     boolean  NOT NULL DEFAULT true,
  CHECK (streak_end IS NULL OR streak_end >= streak_start)
);
```

## 4. 구현 우선순위

| 순서 | 기능 | 이유 |
|------|------|------|
| 1 | 전일 대비 변화 (Delta) | 뷰 이미 존재, 프론트 카피만 추가하면 됨 |
| 2 | 위기 경보 시스템 | 기존 `status` 필드 활용, 시각적 임팩트 큼 |
| 3 | 연속 순위 기록 (Streaks) | 테이블 생성 + 스크래핑 로직 수정 필요, 데이터 축적 후 효과 극대화 |
